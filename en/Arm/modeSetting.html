1. Mode settings

1.1 Get the list of delay rules for all modes

Interface Description:

Get a list of delay rules for all modes

- (void)getDeviceListWithMode:(ArmModeStatus)modeStatus
                       homeId:(long long)homeId
                      success:(GetArmedRuleDeviceList)success
                      failure:(TYFailureError)failure;

Parameter Description:

parameter Description
homeId Family id
success Success callback
failure Failure callback
modeStatus target mode

Data Model

Data field type of data Description
mode enum(ArmModeStatus) Arming mode "Currently only arming at home and arming away from home"
uid NSInteger Arming delay time
ownerId NSInteger Alarm delay time

Objc

     getAllModeDelayRuleWithHomeId:[self getHomeId]
     success:^(NSArray<TYArmedRuleDelayTimeModel *> * _Nullable delayTimeList) {

    } failure:^(NSError *error) {

    }];

1.4 Get a list of device rules in a specified mode

Interface Description:

Get the list of device rules in the specified mode

/// Gets the device in the specified deployment mode
///
/// @param modeStatus arm mode(stay, away)
/// @param homeId  Home ID
/// @param success Success Callback
/// @param failure Failure Callback
- (void)getDeviceListWithMode:(ArmModeStatus)modeStatus
                       homeId:(long long)homeId
                      success:(GetArmedRuleDeviceList)success
                      failure:(TYFailureError)failure;

Parameter Description:

parameter Description
modeStatus Arming mode (arming at home, arming away from home)
homeId Family id
success Success callback
failure Failure callback

Data Model

数据字段 数据类型 说明
deviceId NSString * Device Id
uid NSString * User id
ownerId NSString * Location owner id
name NSString * Device name
customName NSString * Deivce nickName
ability NSInteger Device ability
productId NSString * Device product Id
productCode NSString * Device product Code
categoryName NSString * Device category name
sub BOOL Is it a sub-device fixed
fixed BOOL Is it an environmental device (Environmental devices do not participate in the selection, all arming types are involved in linkage)
selected BOOL Whether selected, whether the security device participates in the rules of the arming type
icon NSString * Device picture
activeTime NSTimeInterval Activation time of the device

Code example:

Objc

[[TuyaSecurityArmedRuleSetting sharedInstance]
     getDeviceListWithMode:(ArmModeStatus)modeType
     homeId:homeId
     success:^(NSArray<TYArmedRuleDeviceModel *> * _Nullable nDeviceList) {

    }
     failure:^(NSError *error) {

    }];
