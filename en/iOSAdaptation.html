iOS Version Adaptation

During the development process, you can turn on the Debug mode and print some logs for problem analysis.

iOS 13 adaptation

Wi-Fi permission changes

Starting from iOS 13, if the geographic location permission is not turned on, [[TuyaSmartActivator sharedInstance] currentWifiSSID]the correct SSID will not be obtained (provided that the Wi-Fi permission is turned on). In this case, the system will return to WLAN or Wi-Fi by default. The following is Apple Official email description

As we announced at WWDC19, we're making changes to further protect user privacy and prevent unauthorized location tracking. Starting with iOS 13, the CNCopyCurrentNetworkInfo API will no longer return valid Wi-Fi SSID and BSSID information. Instead, the information returned by default will be: 

SSID: “Wi-Fi” or “WLAN” (“WLAN" will be returned for the China SKU)
BSSID: "00:00:00:00:00:00"
  1. Confirm that the app has enabled geographic location permissions
  2. Confirm that the BSSID obtained through the system method is 00:00:00:00:00:00, it is considered to be the default return of the system. The result is not available and requires additional processing by the developer, such as manually entering the Wi-Fi name.

Bluetooth permission changes

If you use the SDK to develop an integrated Tuya Bluetooth device, you must pay attention to the new "App Bluetooth permission" added in iOS 13. In the iOS 13 system, in addition to the system Bluetooth permission, each App will have its own Bluetooth permission. If Bluetooth is used in the app, each app will ask whether to allow the app to use Bluetooth permission before launching for the first time.

In iOS 13, the original Apple Bluetooth application for permission to use the NSBluetoothPeripheralUsageDescription field, replaced by NSBluetoothAlwaysUsageDescription fields. Add new fields in info.plist.

If you are compiling and using SDK to develop mesh functions in Xcode 11, the device may not be searched. You need to TuyaSmartBLEMeshKitupgrade to 2.12.47 or later, this version has been made fit; if other Xcode version, no update

When the user chooses not to allow it, it will affect the use of the Bluetooth function. At this time, it is recommended to guide the user to turn on the Bluetooth permission (the status of the Bluetooth center at this time is CBCentralManagerStateUnauthorized )

iOS 12 adaptation

iOS 12 use [[TuyaSmartActivator sharedInstance] currentWifiSSID] can not get to SSID

To obtain WiFi information in Xcode 10, you need to enable relevant permissions. The solution:

  • Xcode -> [Project Name] -> Targets -> [Target Name] -> Capabilities -> Access WiFi Information -> ON

Just open the above permissions.
