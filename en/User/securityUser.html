1.3 User uid registration and login (Already have an account system)

Interface Description

Registration and login are an integrated interface. If you are registered, you will automatically log in. If you are not registered, you will automatically register and log in.

/// 使用uid登录注册
/// uid login or register users
///
/// @param countryCode 国家码 / countryCode
/// @param uid 三方的唯一识别ID / Union identify ID
/// @param password  密码 / password
/// @param success 成功回调 / Success Callback
/// @param failure 失败回调 / Failure Callback
- (void)loginOrRegisterWithCountryCode:(NSString *)countryCode
                                   uid:(NSString *)uid
                              password:(NSString *)password
                               success:(TYSuccessID)success
                               failure:(TYFailureError)failure;

Parameter Description

参数 说明
countryCode 国家码
uid 三方的唯一识别ID
password 密码
success 成功回调
failure 失败回调

Code example

Objc

[[TuyaSmartSecurityUser sharedInstance] loginOrRegisterWithCountryCode:@"your_country_code" uid:@"your_uid" password:@"your_password" success:^(id result) {
  NSLog(@"loginOrRegisterWithCountryCode success: %@", result);
} failure:^(NSError *error) {
  NSLog(@"loginOrRegisterWithCountryCode failure: %@", error);
}];

Swift:

TuyaSmartSecurityUser.sharedInstance()?.loginOrRegisterWithCountryCode("your_country_code", uid: "your_uid", password: "your_password", success: { 
  (result) in
    print("loginOrRegisterWithCountryCode success: \(result)")
}, failure: { 
  (error) in 
    if let e = error {
        print("loginOrRegisterWithCountryCode failure: \(e)")
  }
})

2 User Service

2.1 User service synchronization

After successful registration or login through the 1.1.x interface, the user's service status needs to be verified. Call getServiceInfoWithHandler:failure: immediately to check the status of service binding;

Two situations related to user service:

  1. Use security SaaS default binding: Security SaaS configures a service provider for customers by default in the background, and all user information will be collected on this SaaS platform without distinction

  2. Security SaaS is not the default binding: the need to log in to your security SaaS background created activation code, by getServiceInfoWithHandler: failure: the returned data model, check whether the binding, if not binding, then bindServiceWithCode: Success: failure: ; In this way, the user’s channel can be distinguished by the activation code

Interface Description

Obtain and synchronize user service information

Obtain the service provider ID, channel ID, and activation code bound to the user, and determine whether the customer is bound to the activation code

If you don’t need the activation code, you can directly contact Tuya Development to enable the default activation code to ensure that the value obtained every time is bound.


/// 获取用户的服务信息
/// get user service information.
///
/// 如果用户已经绑定激活码，当前服务信息会记录在本地
/// If the user has already bound the activation code, the current service information is recorded locally
///
/// @param success 成功回调 / Success Callback
/// @param failure 失败回调 / Failure Callback

- (void)getServiceInfoWithHandler:(GetUserServiceHandler)success
                          failure:(TYFailureError)failure;

Parameter Description

参数 说明
success 成功回调
failure 失败回调

Return value description:

返回值 类型 说明
isBind NSInteger 是否绑定激活码
code NSString * 激活码
channelId NSString * 绑定的激活码对应的渠道ID
dealerId NSString * 绑定的激活码对应的服务商ID

Code example:

Objc

[[TuyaSmartSecurityUser sharedInstance] getServiceInfoWithSuccess:^(TuyaSmartSecurityUserServiceModel * _Nullable userServiceModel) {
        if (userServiceModel.isBind) {
            NSLog(@"success");
        } else {
            NSLog(@"need to bind code");
        }
    } failure:^(NSError *error) {
        NSLog(@"error:  %@ ", error)
    }];

Swift

TuyaSmartSecurityUser.sharedInstance().getServiceInfo(success: { (userModel) in
            if (userModel?.isBind == 1) {
                print("success")
            } else {
                print("need to bind dealer")
            }
        },failure: { (error) in
            if let error = error {
                print("show error: /(error))")
            }
        })

2.2 Bind activation code

Interface Description

Customers can distinguish users by channel based on the activation code created by themselves


/// 绑定服务激活码
/// Binding service activation code
///
/// @param aCode 激活码 / Activation code
/// @param success 成功回调 / Success Callback
/// @param failure 失败回调 / Failure Callback
- (void)bindServiceWithCode:(NSString *)aCode
                    success:(TYSuccessID)success
                    failure:(TYFailureError)failure;

Parameter Description

参数 说明
aCode 激活码
success 成功回调
failure 失败回调

Code example:

Objc:

[[TuyaSmartSecurityUser sharedInstance] bindServiceWithCode:@"your code" success:^(id result) {
        if (result) {
            NSLog(@"bind code success");
        }
} failure:^(NSError *error) {
        NSLog(@"error:  %@ ", error);
}];

Swift:

TuyaSmartSecurityActivationCode.sharedInstance().bindService(withCode: "your code", success: { (result) in
      if (result != nil) {
             print("bind code success.")
      }
},failure: { (error) in
      if let error = error {
             print("show error: /(error))")
       }
})
