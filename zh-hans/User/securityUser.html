1.3 用户 uid 注册和登录（已经拥有账号体系）

接口说明：

注册和登录为一体的接口，如果注册了就自动登录，如果没有注册就自动注册并且登录。

/// 使用uid登录注册
/// uid login or register users
///
/// @param countryCode 国家码 / countryCode
/// @param uid 三方的唯一识别ID / Union identify ID
/// @param password  密码 / password
/// @param success 成功回调 / Success Callback
/// @param failure 失败回调 / Failure Callback
- (void)loginOrRegisterWithCountryCode:(NSString *)countryCode
                                   uid:(NSString *)uid
                              password:(NSString *)password
                               success:(TYSuccessID)success
                               failure:(TYFailureError)failure;

参数说明：

参数 说明
countryCode 国家码
uid 三方的唯一识别ID
password 密码
success 成功回调
failure 失败回调

代码示例

Objc

[[TuyaSmartSecurityUser sharedInstance] loginOrRegisterWithCountryCode:@"your_country_code" uid:@"your_uid" password:@"your_password" success:^(id result) {
  NSLog(@"loginOrRegisterWithCountryCode success: %@", result);
} failure:^(NSError *error) {
  NSLog(@"loginOrRegisterWithCountryCode failure: %@", error);
}];

Swift:

TuyaSmartSecurityUser.sharedInstance()?.loginOrRegisterWithCountryCode("your_country_code", uid: "your_uid", password: "your_password", success: { 
  (result) in
    print("loginOrRegisterWithCountryCode success: \(result)")
}, failure: { 
  (error) in 
    if let e = error {
        print("loginOrRegisterWithCountryCode failure: \(e)")
  }
})

2 用户服务

2.1 用户服务同步

通过 1.1.x 的接口注册成功或者登录成功后，需要校验user的服务状态。立即调用getServiceInfoWithHandler:failure:来检查服务绑定的情况；

用户服务相关的两种情况：

1.使用安防SaaS默认绑定：安防SaaS在后台默认配置了一个服务商给客户，所有的用户信息会全部收集在这个SaaS平台上，并且不区分；

2.不做安防SaaS默认绑定: 需要自己登录安防SaaS后台创建激活码，通过getServiceInfoWithHandler:failure:返回的数据model，校验是否绑定，如果没有绑定，然后通过bindServiceWithCode:success:failure:;这种方式，可以通过激活码来区分用户的渠道

接口说明

获取并同步用户的服务信息

获取用户绑定的服务商Id，渠道ID，绑定的激活码，并且可以判断出客户是否绑定激活码

如果不需要激活码的操作，直接联系 Tuya 开发开启默认激活码，这样保证每次获取到的值都是绑定过的


/// 获取用户的服务信息
/// get user service information.
///
/// 如果用户已经绑定激活码，当前服务信息会记录在本地
/// If the user has already bound the activation code, the current service information is recorded locally
///
/// @param success 成功回调 / Success Callback
/// @param failure 失败回调 / Failure Callback

- (void)getServiceInfoWithHandler:(GetUserServiceHandler)success
                          failure:(TYFailureError)failure;

参数说明：

参数 说明
success 成功回调
failure 失败回调

返回值说明：

返回值 类型 说明
isBind NSInteger 是否绑定激活码
code NSString * 激活码
channelId NSString * 绑定的激活码对应的渠道ID
dealerId NSString * 绑定的激活码对应的服务商ID

代码示例：

Objc

[[TuyaSmartSecurityUser sharedInstance] getServiceInfoWithSuccess:^(TuyaSmartSecurityUserServiceModel * _Nullable userServiceModel) {
        if (userServiceModel.isBind) {
            NSLog(@"success");
        } else {
            NSLog(@"need to bind code");
        }
    } failure:^(NSError *error) {
        NSLog(@"error:  %@ ", error)
    }];

Swift

TuyaSmartSecurityUser.sharedInstance().getServiceInfo(success: { (userModel) in
            if (userModel?.isBind == 1) {
                print("success")
            } else {
                print("need to bind dealer")
            }
        },failure: { (error) in
            if let error = error {
                print("show error: /(error))")
            }
        })

2.2 绑定激活码

接口说明：

客户可以根据自己创建的激活码，对于用户进行渠道区分


/// 绑定服务激活码
/// Binding service activation code
///
/// @param aCode 激活码 / Activation code
/// @param success 成功回调 / Success Callback
/// @param failure 失败回调 / Failure Callback
- (void)bindServiceWithCode:(NSString *)aCode
                    success:(TYSuccessID)success
                    failure:(TYFailureError)failure;

参数说明：

参数 说明
aCode 激活码
success 成功回调
failure 失败回调

代码示例：

Objc:

[[TuyaSmartSecurityUser sharedInstance] bindServiceWithCode:@"your code" success:^(id result) {
        if (result) {
            NSLog(@"bind code success");
        }
} failure:^(NSError *error) {
        NSLog(@"error:  %@ ", error);
}];

Swift:

TuyaSmartSecurityActivationCode.sharedInstance().bindService(withCode: "your code", success: { (result) in
      if (result != nil) {
             print("bind code success.")
      }
},failure: { (error) in
      if let error = error {
             print("show error: /(error))")
       }
})
